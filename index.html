NewsBlur requires the API Proxy below to be enabled. Configure the proxy URL and any auth headers your proxy needs (e.g. session cookie, API key) there.

Read-it-later Accounts

Uploads the article content already fetched by NRP (plus any of its images already in NRP's cache) instead of letting Readeck re-fetch and extract the page itself — useful when a site's images need the same login/session as the article and would otherwise fail Readeck's own later fetch. Falls back automatically to sending just the URL and title if the upload fails.

Cache

These cap how much of a response NRP will actually download before giving up — checked against the response's reported size up front, and against how much has actually arrived as it streams in, so an oversized file gets abandoned mid-download instead of being fetched in full and only then discarded for being too big to cache.

Clearing removes all cached articles, images, thumbnails, and pending sync actions, but keeps your selected feeds so the next sync starts fresh.

Logging

A lightweight diagnostic log for troubleshooting: images too large to cache in full (re-fetched from the network each time they're shown, though thumbnails for these are still cached where possible), plus thumbnail-generation tracing and other app events.

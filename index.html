Read-it-later Accounts

Uploads the article content already fetched by NRP (plus any of its images already in NRP's cache) instead of letting Readeck re-fetch and extract the page itself — useful when a site's images need the same login/session as the article and would otherwise fail Readeck's own later fetch. Falls back automatically to sending just the URL and title if the upload fails.

Appearance

Only syncs while NRP is actually open and visible in this tab — not a true background sync, since that needs OS/browser support this app doesn't rely on. If NRP is fully closed or backgrounded, nothing syncs until you reopen it or pull to refresh.

Cache

These cap how much of a response NRP will actually download before giving up — checked against the response's reported size up front, and against how much has actually arrived as it streams in, so an oversized file gets abandoned mid-download instead of being fetched in full and only then discarded for being too big to cache.

Runs automatically as part of every sync. Only ever touches read, unstarred articles — unread and starred articles are never purged, regardless of age or these limits. For Miniflux/NewsBlur, purged articles are deleted outright — the server itself only ever reports unread items, so a deleted one can't resurface as unread. For local feeds, a small placeholder is kept instead, since there's no server-side read tracking and a feed could re-list the same old item later; the setting above controls how long that placeholder sticks around before being fully removed too.

Cached: —

Clearing removes all cached articles, images, thumbnails, and pending sync actions, but keeps your selected feeds so the next sync starts fresh.

Export/Import

A full backup: every account (including credentials), every selected feed with its folder, and your appearance/cache settings. Meant for restoring or migrating NRP itself to a new device — import replaces matching settings wholesale rather than merging. Since it includes credentials, treat the exported file like a password. For exporting or importing an individual account's feed list instead, edit that account under Reader Accounts above — OPML export/import lives there now.

Logging

A lightweight diagnostic log for troubleshooting: images too large to cache in full (re-fetched from the network each time they're shown, though thumbnails for these are still cached where possible), plus thumbnail-generation tracing and other app events.

Pending sync actions: —

Read/unread, star, and send-to-read-it-later actions that couldn't reach the server yet (offline, or the server rejected them) wait here and retry automatically, up to 15 attempts before being given up on and logged above. Clearing removes them without retrying further — your local read/unread/starred state is unaffected either way, only whether it's ever reflected on the server.

Security

Requires a password to open NRP at all — the entire stored configuration (accounts, credentials, feeds, settings) is encrypted as a single file, and nothing renders until it's unlocked. Protects against another user of this device opening the tab and browsing your feeds, and against casual exposure of an exported backup file. It does not protect against a compromised device, where JavaScript could read the password as you type it. Off by default. The password itself is never stored — if you forget it, there's no way to recover what's encrypted; the lock screen's Reset option is the only way back in, and it erases everything, not just the password.

Without this, expect to be asked for your password surprisingly often — mobile browsers (Android especially) frequently reload backgrounded tabs, and each reload otherwise means unlocking again. This caches the derived key in this tab's session storage (cleared when the tab/browser fully closes, separately expires after an hour either way) rather than the password itself — a real convenience/security tradeoff, not a free lunch: anyone with access to this tab's dev tools during that window could extract the cached key. On by default.
